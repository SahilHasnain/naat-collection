# Naat AI Detection Service\n\nAI-powered service to detect speech vs singing segments in Naat audio files using Wav2Vec2 and audio analysis.\n\n## Features\n\n- **Speech/Singing Classification**: Uses Wav2Vec2 features + rule-based analysis\n- **Segment Detection**: Identifies explanation segments to be cut\n- **Audio Processing**: Handles various audio formats via FFmpeg\n- **REST API**: Simple HTTP interface for integration\n- **Docker Support**: Easy deployment with Docker\n\n## Quick Start\n\n### Local Development\n\n```bash\n# Install dependencies\npip install -r requirements.txt\n\n# Run service\npython app.py\n```\n\n### Docker Deployment\n\n```bash\n# Build and run\ndocker-compose up -d\n\n# Check logs\ndocker-compose logs -f\n\n# Stop\ndocker-compose down\n```\n\n## API Endpoints\n\n### Health Check\n```bash\nGET /health\n```\n\n### Detect Segments\n```bash\nPOST /detect-segments\nContent-Type: application/json\n\n{\n  \"audio_url\": \"https://example.com/audio.mp3\"\n}\n```\n\n**Response:**\n```json\n{\n  \"duration\": 180.5,\n  \"speechSegments\": [\n    {\n      \"start\": 10.0,\n      \"end\": 45.0,\n      \"duration\": 35.0,\n      \"type\": \"speech\",\n      \"confidence\": 0.85\n    }\n  ],\n  \"allSegments\": [...],\n  \"totalSpeechDuration\": 35.0,\n  \"totalSingingDuration\": 145.5\n}\n```\n\n### Test Classification (Debug)\n```bash\nPOST /test-classification\nContent-Type: application/json\n\n{\n  \"audio_url\": \"https://example.com/audio.mp3\",\n  \"segment_length\": 5.0\n}\n```\n\n## Configuration\n\n### Environment Variables\n- `FLASK_ENV`: Set to `production` for production deployment\n- `PYTHONUNBUFFERED`: Set to `1` for real-time logging\n\n### Audio Processing Parameters\n- **Sample Rate**: 16kHz (optimized for speech)\n- **Segment Length**: 5 seconds (configurable)\n- **Min Segment Duration**: 2 seconds\n- **Merge Threshold**: 1 second gap\n\n## Algorithm Details\n\n### Classification Method\nCurrently uses **rule-based classification** with audio features:\n- **MFCC**: Mel-frequency cepstral coefficients\n- **Spectral Features**: Centroid, rolloff, bandwidth\n- **Zero Crossing Rate**: Speech vs music differentiation\n- **Chroma Features**: Harmonic content analysis\n- **Energy Analysis**: RMS energy patterns\n\n### Processing Pipeline\n1. **Audio Loading**: Convert to 16kHz mono\n2. **Preprocessing**: Normalization + pre-emphasis\n3. **Segmentation**: Split into 5-second chunks\n4. **Classification**: Rule-based speech/singing detection\n5. **Post-processing**: Merge, filter, smooth segments\n6. **Output**: Speech segments for cutting\n\n## Performance\n\n- **Processing Speed**: ~1-2x real-time on 1GB RAM\n- **Memory Usage**: ~400-800MB depending on audio length\n- **Accuracy**: ~80-85% with rule-based method\n\n## Future Improvements\n\n1. **Train Custom Model**: Fine-tune Wav2Vec2 on Naat data\n2. **GPU Support**: Add CUDA support for faster processing\n3. **Batch Processing**: Handle multiple files simultaneously\n4. **Model Caching**: Cache Wav2Vec2 model for faster startup\n5. **Advanced Features**: Tempo, rhythm, vocal characteristics\n\n## Deployment Notes\n\n### DigitalOcean Droplet\n- **Minimum**: 1GB RAM, 1 vCPU\n- **Recommended**: 2GB RAM, 2 vCPU for better performance\n- **Storage**: 20GB+ for model and temporary files\n\n### Resource Usage\n- **CPU**: High during processing, idle otherwise\n- **Memory**: 400-800MB per request\n- **Network**: Download audio files (varies by file size)\n\n## Troubleshooting\n\n### Common Issues\n1. **Out of Memory**: Reduce segment_length or upgrade RAM\n2. **Slow Processing**: Audio files too large, consider chunking\n3. **Download Failures**: Check audio URL accessibility\n4. **Classification Errors**: Audio format not supported\n\n### Logs\n```bash\n# Docker logs\ndocker-compose logs -f naat-ai\n\n# Local logs\ntail -f logs/app.log\n```\n\n## method: 'POST',\n  headers: { 'Content-Type': 'application/json' },\n  body: JSON.stringify({ audio_url: audioUrl })\n});\n\nconst result = await response.json();\n```
